Builds the design matrix from currently supplied data and stores the result in the Workspace.DesignMatrix property.

The parameters searchRange and fwhmRange are being multiplied, so they must be both nonzero.

Parameters
searchRangeOptional parameter, that sets the range around the centre of mass of a cluster, in which the design matrix values for that cluster will be generated.
fwhmRangeOptional parameter, that sets the range as a function of FWHM around the centre of mass of a cluster, in which the design matrix values for that cluster will be generated.

void IsotopeFit.Workspace.CorrectBaseline ( double []  xAxis = null,
double []  yAxis = null 
)

Calculates baseline corrected signal from raw signal data and baseline correction points. Stores the result in the Workspace.SpectralData.SignalAxis property.

This method uses the PCHIP interpolation to obtain the baseline values, which are stored in the Workspace.SpectralData.Baseline property.

Optional arguments xAxis and yAxis are meant to ease the process of loading data to the Workspace. If not supplied, the funcion will use previously stored values.

Parameters
xAxisOptional x-axis for the baseline correction.
yAxisOptional y-axis for the baseline correction.
Exceptions
WorkspaceExceptionThrown when some of the required data have not been loaded in the Workspace.

◆ CorrectMassOffset()

void IsotopeFit.Workspace.CorrectMassOffset ( Interpolation.Type  interpType,
int  order = -1,
bool  autoCrop = false,
double []  comList = null,
double []  massOffsetList = null 
)

Calculates mass axis corrected for mass offset from previously supplied calibration data and stores the result in the Workspace.SpectralData.MassAxis property.

In case of polynomial inerpolation, the order is considered the highest exponent value of the desired interpolating polynomial, e.g. order = 2 will produce quadratic polynomial.

Optional arguments comList and massOffsetList are meant to ease the process of loading data to the Workspace. If not supplied, the method will use previously stored values.

Parameters
interpTypeType of the interpolation to be used.
orderOrder of the polynomial interpolation. Is marked as optional, but is required if polynomial interpolation is selected.
autoCropOptional flag that specifies, if the automatic cropping of the mass axis should be performed.
comListOptional array of centre-of-mass points to be used for the correction.
massOffsetListOptional array of mass offset points to be used for the correction.
Exceptions
WorkspaceExceptionThrown when some of the required data have not been loaded in the Workspace or their lengths are different.

◆ ResolutionFit()

void IsotopeFit.Workspace.ResolutionFit ( Interpolation.Type  t,
int  order,
double []  comList = null,
double []  resolutionList = null 
)

Fits the previously supplied resolution calibration data and stores the calibration results in the Workspace.ResolutionInterpolation property.

In case of polynomial inerpolation, the order is considered the highest exponent value of the desired interpolating polynomial, e.g. order = 2 will produce quadratic polynomial. Optional arguments are meant to be used by GUI to ease the process of loading data to the Workspace. If not supplied, the method will use previously stored values.

Parameters
tType of the interpolation to use.
orderOrder of the polynomial interpolation. This is relevant only for the polynomial interpolation.
comListOptional array of centre-of-mass points to be used for the calibration.
resolutionListOptional array of resolution points to be used for the calibration.
Exceptions
WorkspaceExceptionThrown when some of the required data have not been loaded in the Workspace or their lengths are different.
